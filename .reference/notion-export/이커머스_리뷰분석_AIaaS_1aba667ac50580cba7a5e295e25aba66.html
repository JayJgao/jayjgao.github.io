3. 요약 작성: 한국어 문법에 맞고 자연스러운 흐름을 유지하면서 다음의 조건에 맞게 세 개의 문장을 작성합니다:
   - 총 150자 내외
   - 각 문장은 50자 내외
   - '-요' 종결어미 사용
   - 부정적 표현과 브랜드명 제외

   - 단순 단어나 문장 전체는 제외
   - 요약문에서 정확히 일치하는 표현만 선택

  • 사용 Query String (Essential)
    • 전체 사용가능 Query string이 아닌, 최소 수준 구현 시 사용될 핵심 쿼리입니다.
    • mall_product_id : 상품의 ID
      • type : String
      • example : 12006590
    • opinion_attribute : 의견 속성 필터
      • type : String
      • example : 소재
    • opinion_group : 의견 그룹 필터
      • type : String
      • example : 소재가 두꺼움
    • order_by : 리뷰의 정렬 순서
      • type : String(enum)
      • enums
        • recommend : 추천순
        • registered_desc : 최신순
        • like_count_desc : 좋아요 높은순
        • satisfaction_desc : 만족도(별점, 평점) 높은순
        • satisfaction_asc : 만족도(별점, 평점) 낮은순
        • satisfaction_desc_with_registered_desc : 평점높은순&최신순
        • satisfaction_asc_with_registered_desc : 평점낮은순&최신순
    • satisfaction_score_weights : 만족도 점수 가중치
      • type : Float (0 이상)
      • example : 2.2
    • offset , limit : 데이터 제어에 사용
      • type: Int
      • exmaple : offset : 0, limit : 20

<em class="negative">부정적인 멘트</em>
    • 응답 데이터 리스트에서 text , sentiment 를 확인할 수 있습니다.
    • text 는 속성이 있는 텍스트의 실제 텍스트 그대로 둔것으로, 리뷰의 내용인 content 와 직접 매칭되는 부분을 하이라이트하는데에 사용할 수 있습니다.
    • sentiment 는 text 에 내용이 부정적(NEGATIVE)인지, 긍정적(POSITIVE)인지, 중립적(NEUTRAL)인지 반환하는 내용입니다.
      • sentiment 를 활용한 리뷰 내용 하이라이트 예시
        • 바지 스커트 정장핏으로 너무 잘 어울려요 오피스룩으로 입기에는 별로에요.

통합EP 신규 스키마 설계 예시

EP File Format

  • file type: tsv
    • seperator: \t
    • file encoding: utf-8
    • header: 첫번째 row는 header로 사용

EP File 이름 규칙

  • 일반 파일 규칙
    • products-{ep_type}-{file_type}-{yyyymmddHHMM}-{index}.tsv
  • 압축 할 경우 규칙
    • products-{ep_type}-{file_type}-{yyyymmddHHMM}-{index}.tsv.tar.gz
  • {ep_type}
    • all
    • summary
  • {file_type}
    • base
  • {yyyymmddHHMM}
    • example: 202312270000
  • {index}
    • 파일 사이즈가 클 경우 분할
    • 0~99 까지 순차적으로 붙여줌
    • 100개까지 전부 나눌필요 없음
    • 파일을 나누지 않을 경우 0번째 파일만 생성

상품 EP 파일 이름규칙 예제

  • 전체: products-all-base-{yyyymmdd}-{index}.jsonl
    • example: 전체 기초정보 파일을 5개로 분할한 경우
      • products-all-base-20231227-0.tsv
      • products-all-base-20231227-1.tsv
      • products-all-base-20231227-2.tsv
      • products-all-base-20231227-3.tsv
      • products-all-base-20231227-4.tsv
  • 요약: products-summary-base-{yyyymmddHHMM}-{index}.tsv
    • example
    • products-summary-base-202312270100-0.tsv

상품 EP 파일 압축 지원

  • 지원 가능한 포맷: tar.gz
  • 압축 시 각 파일별로 압축
  • example: 전체 파일이 2개씩 만들어진 경우 각 파일별로 압축
    • products-all-base-202312270000-0.tsv.tar.gz
    • products-all-base-202312270000-1.tsv.tar.gz
