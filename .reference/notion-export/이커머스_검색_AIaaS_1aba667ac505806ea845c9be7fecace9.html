검색EP 가이드 샘플

EP File Format

  • file type: jsonl
    • format guideline: https://jsonlines.org/
    • file encoding: utf-8

EP File 이름 규칙

  • 일반 파일 규칙
    • products-{ep_type}-{file_type}-{yyyymmddHHMM}-{index}.jsonl
  • 압축 할 경우 규칙
    • products-{ep_type}-{file_type}-{yyyymmddHHMM}-{index}.jsonl.tar.gz
  • {ep_type}
    • all
    • summary
  • {file_type}
    • base
  • {yyyymmddHHMM}
    • example: 202312270000
  • {index}
    • 파일 사이즈가 클 경우 분할
    • 0~99 까지 순차적으로 붙여줌
    • 100개까지 전부 나눌필요 없음
    • 파일을 나누지 않을 경우 0번째 파일만 생성

상품 EP 파일 이름규칙 예제

  • 전체: products-all-base-{yyyymmdd}-{index}.jsonl
    • example: 전체 기초정보 파일을 5개로 분할한 경우
      • products-all-base-20231227-0.jsonl
      • products-all-base-20231227-1.jsonl
      • products-all-base-20231227-2.jsonl
      • products-all-base-20231227-3.jsonl
      • products-all-base-20231227-4.jsonl
  • 요약: products-summary-base-{yyyymmddHHMM}-{index}.jsonl
    • example
    • products-summary-base-202312270100-0.jsonl

상품 EP 파일 압축 지원

  • 지원 가능한 포맷: tar.gz
  • 압축 시 각 파일별로 압축
  • example: 전체 파일이 2개씩 만들어진 경우 각 파일별로 압축
    • products-all-base-202312270000-0.jsonl.tar.gz
    • products-all-base-202312270000-1.jsonl.tar.gz
