## 🚧 Problem
기존 검색엔진을 OpenAPI 기반 SaaS로 전환해야 했고, 이커머스 플랫폼별 데이터 규격 차이와 트래픽 증가 대응이 동시에 필요했습니다.

## 💡 Approach
검색 EP 스키마를 표준화하고, API 계약과 운영 백오피스를 함께 설계해 고객사 신뢰성과 확장성을 같이 확보했습니다.

## ⚙️ Execution
- EP 파일 포맷/명명 규칙/검증 정책을 표준 문서로 정리
- 검색 API 스키마를 고객 사용 기능 중심으로 확장
- 온보딩/운영 상태를 볼 수 있는 관리 화면을 도입
- AI Research/BE/Data Engineering/Product Design과 스크럼 운영

![이커머스 검색 AIaaS 화면](/assets/images/projects/buzzni/buzzni-search-aiaas-shot-01.webp)

## ✅ Results
- 3개 고객사 검색 서비스 납품 (쇼핑엔티, SK스토아, 어바웃펫)
- 7개 기업 PoC 운영으로 세일즈 파이프라인 확장
- 검색 데이터 계약 표준화로 신규 고객 온보딩 속도 개선
