## 🚧 Problem
자연어 스크립트를 Unreal 결과물로 연결하는 구간에서 병목이 컸고, 모듈 계약이 느슨해 실험 실패 비용이 높았습니다.

## 💡 Approach
- 오케스트레이션 레이어와 실행 레이어를 분리했습니다.(S2M=오케스트레이션 레이어)
- 모듈간 데이터 입력/출력 스키마를 확정해 제품 파이프라인을 확립했습니다.

## ⚙️ Execution
- 파이프라인 실패 로그를 기준으로 병목 3개 구간을 먼저 정리
- Scene/Action/Camera/Map/Character 계약을 JSON schema 중심으로 재작성
- LLM 평가셋과 E2E 체크리스트를 운영 루프에 고정할 수 있도록 설계 (LLM evals)
- LangChain·LangGraph 기반 Film Agent 오케스트레이션 레이어 설계에 참여

## ✅ Results
- 합류 3개월 내 첫 LLM-to-Unreal E2E 통합
- 실험 리드타임 약 50% 단축 (600s → 300s)
- MOAI/A2P 확장을 위한 재사용 가능한 베이스 확보 (json 형태의 프로젝트 파일)
- 2026년 1월 CineV 공식 런칭 AI 파트 리딩 기반 마련
