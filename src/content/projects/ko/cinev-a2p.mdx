## 🚧 Problem
외부 생성형 모델을 제품에 붙일 때마다 계약/인프라/운영 정책을 다시 맞춰야 해
적용 리드타임이 주 단위로 늘어졌습니다.

## 💡 Approach
모델별 개별 연동 대신, 라우팅 레이어 + 공통 계약(API/스키마) 기반의
A2P(Application-to-Provider) 통합 구조를 설계했습니다.

## ⚙️ Execution
- I2V / I2I / TTS / Music Gen 공통 입력 스키마 정의
- Provider Adapter 계층을 분리해 모델 교체 비용 축소
- 모델 성능/비용/지연시간 기준의 라우팅 규칙 테이블 운영
- 실패 fallback 및 버전 정책을 배포 파이프라인에 반영

![A2P thumbnail](/assets/images/projects/cinev/a2p-thumb.png)

## ✅ Results
- 외부 모델 적용 사이클: 주 단위 -> 즉시 배포 수준으로 단축
- 신규 모델 온보딩 절차 표준화
- CineV 런칭 구간에서 모델 확장 속도와 운영 안정성 동시 확보
